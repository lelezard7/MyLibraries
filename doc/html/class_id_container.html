Класс IdContainer

Контейнер с настраиваемым способом выдачи ID. Подробнее...

#include <IdManagement.h>

Подробное описание

Контейнер с настраиваемым способом выдачи ID.

По сути IdContainer это объединение std::list и std::set.
IdContainer хранит все записанные в него уникальные ID и выдает их в порядке возрастания, убывания или добавления.
Если метод выдачи ID равен IdIssuingMethod::Static_Ascending, то ID выдаются в порядке возрастания.
Если метод выдачи ID равен IdIssuingMethod::Static_Descending, то ID выдаются в порядке убывания.
Если метод выдачи ID равен IdIssuingMethod::Dynamic, то ID выдаются в порядке их добавления в контейнер.
По умолчанию метод выдачи ID равен IdIssuingMethod::Dynamic.
Мы можем задать способ выдачи ID в конструкторе при создании и менять через метод setIdIssuingMethod(IdIssuingMethod).

Когда метод выдачи ID равен IdIssuingMethod::Dynamic, все ID хранятся в unorderedIds_, а при IdIssuingMethod::Static_Ascending и IdIssuingMethod::Static_Descending в orderedIds_:

std::list<T> unorderedIds_;
std::set<T> orderedIds_;
Предупреждения
IdContainer не хранит порядок добавления ID, он просто перекидывает все хранимые ID в std::list в случае если способ выдачи ID равен IdIssuingMethod::Dynamic и в std::set если способ выдачи ID равен IdIssuingMethod::Static_Ascending или IdIssuingMethod::Static_Descending.
Отсюда вытекает проблема при которой после смены режима выдачи с IdIssuingMethod::Dynamic на IdIssuingMethod::Static_Ascending или IdIssuingMethod::Static_Descending, а затем обратно на IdIssuingMethod::Dynamic, IdContainer уже не будет выдавать ранее добавленные ID в том порядке в котором мы их добавляли в него. Но вновь добавленные ID будут выдаваться в порядке добавления.
Сейчас это скорее баг нежели фича. В будущих версиях этот момент будет доработан (будет обратно совместим).

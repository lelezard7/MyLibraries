◆ ReservationMethod

enum class ONF::ReservationMethod
strong

Отвечает за способ резервации ID методом RangeIdManager::reserve(T, ReservationMethod).

Каждый перечислитель, несмотря на описание подчиняется всем правилам RangeIdManager.
Это правила резервации ID, поведения границ диапазона и т.д.
Т.е., если нужно передвинуть границу после резервации ID или перепрыгнуть на резервируемый ID, это произойдет.

Подробнее о том, как каждый из перечислителей влияет на RangeIdManager::reserve(T, ReservationMethod) читайте в описании данного метода.

Элементы перечислений
NotSet 

Говорит, что нужно просто зарезервировать указанный ID.

Interpolate 

Говорит, что нужно зарезервировать указанный ID и интерполировать все ID от границы диапазона (включительно) до указанного ID.

ReserveRange 

Говорит, что нужно зарезервировать все ID от границы диапазона (включительно) до указанного ID (включительно).

Возвращает ближайший к переданному ID, стандартный ID.

Под ближайшим стандартным ID подразумевается тот, который находится ближе всего к переданному ID и при этом находится между target и start.
Знак числа шага переданного в step не имеет значения.

В следующем примере для ID равного -4,3, ближайшим стандартным является -4, но не -5.
Для ID равного 5,5, ближайшим стандартным является 5, но не 6.

findNearestStdId(5.5, 0, 1); //result: 5
findNearestStdId(-4.3, 0, 1); //result: -4

◆ isStandardId()

template<class T >
bool ONF::isStandardId ( T  target,
T  start,
T  step 
)

Проверяет, является ID стандартным или нет.

Стандартный ID - это число, которое является \( n \)-м членом арифметической прогрессии.
\( n \)-й член арифметической прогрессии вычисляется по формуле:

\( a_n = a_1 + (n - 1)d \)

Отсюда можно вывести формулу:

\( n = \frac{a_n - a_1}{d} + 1 \)

Где \( d \neq 0 \)

Обозначение:
\( a_n \) - ID, которое мы должны проверить на то, стандартное оно или нет.
\( a_1 \) - начальный ID.
\( d \) - размер шага.
\( n \) - позиция ID в арифметической прогрессии.

Если в результате вычислений, \( n \) окажется целым числом, то проверяемый ID ( \( a_n \)) можно считать стандартным. Если \( n \) окажется дробным, то проверяемый ID будет нестандартным.

Аргументы
target- ID, которое нужно проверить.
start- начальный ID.
step- размер и направление шага.
Возвращает
Если target окажется стандартным ID, вернет true, иначе false.

◆ trimString()

template<class T >
int ONF::trimString ( T &  str,
const decltype(T())&  rule,
unsigned  trimmingSide 
)

Удаляет заданные символы с указанной стороны до тех пор, пока не встретится иной символ.

В str передается строка которую нужно обработать и в которую будет записан результат обработки.
В rule передается строка с символами которые нужно удалить.
В trimmingSide передается перечислитель из перечисления TrimmingSide, который говорит с какой стороны нужно удалять указанные в rule символы.
Если указать в trimmingSide перечислитель TrimmingSide_Left, данная функция начнет перебирать все символы слева направо. Если символ равен одному из символов строки rule, он удаляется, если нет, то операция прерывается, и функция возвращает результаты.
Если указать в trimmingSide перечислитель TrimmingSide_Right, будет происходить тоже что описано выше только символы перебираться будут справа налево.
Если TrimmingSide_Left и TrimmingSide_Right указать вместе, то сперва функция удалит указанные в rule символы слева, а потом справа. Потом функция вернет результаты.

Аргументы
str- строка для обработки. Так же в нее будет записан конечный результат.
rule- символы которые нужно удалить.
trimmingSide- сторона с которой нужно удалить указанные символы.
Возвращает
Возвращает количество удаленных символов.
